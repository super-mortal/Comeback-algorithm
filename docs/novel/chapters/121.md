# 第一百二十一章 三个版本

白板上那行字看了三天，林知行还是没动笔。

"产品架构问题：免费版太好了，好到没人需要付费版。"

他坐在折叠桌前，面前摊着三样东西：用户行为漏斗图、方小满跑来的三家客户反馈、陆可盈要求的方案截止日期——还有四天。

窗外天刚亮，四环外的车流声隐约传进来。方小满还在打呼噜，声音不大，但均匀得像节拍器。

林知行打开笔记本，新建了一个文档，标题打了三个字又删掉。

他想起姜意说过的话：你最大的问题是把用户当变量。

当时他不服。现在他服了。

用户不是变量。用户是人。人不会按你的逻辑行动。

他盯着漏斗图看了很久。注册到试用，89%。试用到付费，6.9%。卡点在第四天——用户试用三天后，发现免费版够用，就走了。

6.9%。

这个数字在他脑子里转。

他打开计算器，算了一笔账：68家注册用户，15家付费。每家月费9.9元，月收入148.5元。服务器费800元，房租1400元，生活费3000元，总支出5200元。

148.5元对5200元。

杯水车薪。

他关掉计算器，又打开用户行为分析页面。这次他不看漏斗图，看单个用户的操作记录。

第一家：奶茶店。注册五天，前三天每天登录四次，用库存预测、定价建议、排班优化。第四天点开决策日志，停留12秒，退出。之后再没登录。

第二家：五金店。注册四天，行为几乎一模一样。前三天高频使用，第四天点开付费功能，消失。

第三家。第四家。第五家。

全是同样的模式。

林知行靠在椅背上，盯着天花板。天花板上那道裂缝还在，从左边延伸到右边，像一条分岔的路。

他忽然想起一个词：**用户价值分层**。

不是功能分层，是价值分层。

他打开一个新文档，开始写：

```
免费版：基础AI建议
- 库存预测（基础）
- 定价建议（基础）
- 排班优化（基础）
目标：让用户"能用"，建立信任
```

写完第一行，他停了。

"能用"是什么意思？

他想了想，继续写：

```
标准版：9.9元/月
- 决策日志（AI解释）
- 历史趋势分析
- 自定义报告
目标：让用户"看懂"，建立依赖
```

写完第二行，他又停了。

"看懂"够吗？

用户要的不是看懂，是效果。

他继续写：

```
专业版：99元/月
- 数据飞轮（持续优化）
- 定制分析（行业专属）
- 效果保证（三个月见效）
目标：让用户"赚钱"，建立壁垒
```

写完第三行，他盯着屏幕看了很久。

三个版本，三层价值。

免费版给能力，标准版给认知，专业版给结果。

用户不是在买功能，是在买效果。

他忽然明白了。

陆可盈说得对：问题不在功能，在架构。

免费版太好了，好到用户觉得够用了。不是因为功能多，是因为用户不知道"更好"是什么样。

如果把"更好"明码标价呢？

如果让用户看到，花9.9元能看到AI在想什么，花99元能让AI帮他赚钱呢？

他站起来，走到白板前，拿起马克笔。

---

方小满是被马克笔的吱呀声吵醒的。

他翻了个身，探出头来，看到林知行站在白板前，正在画一个表格。

"几点了？"他问。

"六点半。"林知行没回头。

"你干嘛呢？"

"设计产品架构。"

方小满打了个哈欠，从床上爬起来，走到白板前。

白板上画着一个三列的表格：

```
免费版          标准版          专业版
基础AI建议      决策日志        数据飞轮
库存预测        历史趋势        定制分析
定价建议        自定义报告      效果保证
排班优化        
```

"这是什么？"方小满问。

"产品分层。"林知行说，"陆可盈说免费版太好了，好到没人需要付费版。所以我要重新设计架构。"

"怎么设计？"

"三层。"林知行说，"免费版给基础功能，让用户能用。标准版给决策日志，让用户看懂。专业版给数据飞轮和定制分析，让用户赚钱。"

方小满盯着表格看了很久。

"免费版的功能砍了？"他问。

"没砍。"林知行说，"库存预测、定价建议、排班优化都在。"

"那标准版多什么？"

"决策日志。"林知行说，"AI解释功能。用户能看到AI为什么这么建议。"

"这个之前就有啊。"方小满说，"就是付费功能。"

"对。"林知行说，"但现在我要把它单独拎出来，作为标准版的核心卖点。不是附属功能，是主打功能。"

"有什么区别？"

"区别在于用户心理。"林知行说，"以前用户觉得决策日志是'额外功能'，可有可无。现在我要让他们觉得，不买标准版就看不到AI在想什么。"

方小满想了想。

"那专业版呢？"他问。

"数据飞轮和定制分析。"林知行说，"这个暂时做不出来。"

"做不出来？"

"对。"林知行说，"数据飞轮需要大量用户数据喂养算法，我们现在只有68家用户，数据量不够。定制分析需要行业know-how，我还没有积累。"

"那专业版怎么卖？"

"先不卖。"林知行说，"先设计出来，等数据和能力够了再上线。"

方小满愣了一下。

"你设计一个做不出来的版本？"他说，"这不是纸上谈兵吗？"

"不是纸上谈兵。"林知行说，"是战略规划。用户需要看到未来，看到这个产品能走到哪里。专业版是目标，不是现在。"

"目标有什么用？"方小满说，"用户要的是现在能用的东西。"

"用户要的是效果。"林知行说，"专业版承诺效果——三个月见效，达不到退款。这是价值保证。"

"你拿什么保证？"方小满说，"我们现在连长沙的试点数据都没有。"

"所以专业版先不卖。"林知行说，"先卖免费版和标准版。等有了标杆客户，再推专业版。"

方小满盯着白板，没说话。

林知行继续说："免费版给基础功能，让用户能用。标准版给决策日志，让用户看懂。专业版给效果保证，让用户赚钱。三层价值，三个价格，三种用户。"

"三种用户？"方小满问。

"对。"林知行说，"免费版给价格敏感的用户，他们要的是基础功能。标准版给愿意付费的用户，他们要的是透明度。专业版给大客户，他们要的是效果。"

"大客户在哪？"

"不知道。"林知行说，"但先设计好架构，等大客户来了，我们能接住。"

方小满想了很久。

"你说的这些，"他说，"我听着像在画饼。"

"不是画饼。"林知行说，"是产品路线图。"

"路线图有什么用？"方小满说，"路线图不能当饭吃。"

"路线图能让投资人看到方向。"林知行说，"陆可盈要的不是一个能赚钱的产品，是一个能讲故事的产品。故事就是路线图。"

方小满愣了一下。

"你什么时候学会讲故事了？"他问。

"跟你学的。"林知行说，"在灵犀的时候，你不是说'用户要的不是最准的回答，是让人觉得被听到了'吗？"

"那是我说的？"

"你说的。"林知行说，"我记住了。"

方小满没接话。

---

两人吃了早饭，继续讨论。

林知行在白板上画了一个功能矩阵，把每个功能标上"免费""标准""专业"的标签。

"库存预测，免费。"他说，"这是基础功能，用户必须能用。"

"决策日志，标准。"他继续说，"这是核心差异，用户必须付费才能看到AI在想什么。"

"数据飞轮，专业。"他最后说，"这是效果保证，用户必须付费才能让算法持续优化。"

方小满盯着矩阵看了很久。

"那定价呢？"他问。

"免费版0元。"林知行说，"标准版9.9元/月，跟之前的付费版一样。专业版99元/月。"

"99元？"方小满说，"谁会付99元？"

"大客户。"林知行说，"比如张老板那种，三十七家分校，排课系统年费两万。对他们来说，99元一个月是小钱。"

"但我们没有张老板那种客户。"方小满说。

"所以专业版先不卖。"林知行说，"先设计好，等客户来了再推。"

方小满又沉默了。

"你这套逻辑，"他说，"我听着像在灵犀的时候听沈渡讲PPT。"

林知行愣了一下。

"什么意思？"

"意思是，"方小满说，"你讲得很好，但我不确定能不能落地。"

"能落地。"林知行说，"免费版功能不变，只是把决策日志单独拎出来作为标准版的核心。技术上不需要改动，只需要调整界面和定价。"

"那两周能做完吗？"

"能。"林知行说。

"你上次也说两周。"方小满说，"排课系统的时候，你说两周，结果做了三周。信用评分系统的时候，你说两周，结果做了一个月。"

林知行没接话。

方小满继续说："我不是不相信你。我是不相信'两周'这个数字。你每次说两周，都是按最理想的情况算的。但实际做起来，总有意外。"

"这次不一样。"林知行说。

"哪里不一样？"

"这次只需要调整界面和定价，不需要写新代码。"林知行说，"功能都在，只是重新包装。"

"重新包装也要时间。"方小满说，"你要改界面、改文档、改定价页面、改用户引导。这些加起来，至少三周。"

"三周也行。"林知行说。

"那陆可盈的方案怎么办？"方小满说，"她要下周看方案。"

"方案先给她。"林知行说，"产品路线图、功能矩阵、定价策略，这些不需要做出来，只需要写出来。"

"写出来就行？"

"对。"林知行说，"她要的是方向，不是产品。"

方小满想了想。

"行。"他说，"那你写方案，我继续跑客户。"

"好。"

"还有一件事。"方小满说。

"什么？"

"你说的'效果保证'，"他说，"三个月见效，达不到退款。这个怎么保证？"

林知行愣了一下。

"我不知道。"他说，"但先写上。等有了标杆客户，再想办法。"

"你这是先画靶再射箭。"方小满说。

"对。"林知行说，"先画靶，再找箭。箭找不到，靶还是得画。"

方小满盯着他看了几秒。

"你变了。"他说。

"哪里变了？"

"以前你不会画靶。"方小满说，"以前你只会写代码。"

林知行没接话。

他走到白板前，拿起马克笔，在矩阵下面写了一行字：

**产品架构：三个版本，三层价值，三种用户。**

然后在下面写了一行小字：

**从工程师到产品经理的第一次尝试。**

---

接下来三天，林知行一直在写方案。

他把产品路线图、功能矩阵、定价策略、用户分层、数据飞轮、效果保证，全都写进了一个十二页的文档里。

方小满看了第一版，说："太技术了，看不懂。"

林知行改了第二版，加了用户故事和场景描述。

方小满看了第二版，说："好一点，但还是太长。投资人没耐心看十二页。"

林知行改了第三版，压缩到六页，只保留核心逻辑和关键数据。

方小满看了第三版，说："这个可以。"

林知行把方案发给陆可盈。

陆可盈当天晚上回了消息，只有一句话：

"方案收到。下周视频会议讨论。"

林知行盯着这条消息，心里松了一口气。

方小满在旁边看了一眼，说："她没说好也没说坏。"

"没说坏就是好。"林知行说。

"你怎么知道？"

"因为如果她觉得不好，会直接说。"林知行说，"她不是那种会委婉的人。"

方小满想了想，点了点头。

"行。"他说，"那我们开始做产品调整？"

"先等她确认。"林知行说，"确认了再动手。"

"万一她不确认呢？"

"那再改。"林知行说。

方小满叹了口气。

"你什么时候变得这么有耐心了？"他问。

"不是耐心。"林知行说，"是谨慎。在灵犀的时候，我越位了一次，被沈渡骂了。现在我学会了——先确认，再动手。"

"在灵犀的时候，你不是这样的。"方小满说。

"那时候我是工程师。"林知行说，"工程师觉得对就做。现在我是产品经理。产品经理要先问用户要不要。"

"用户是谁？"

"陆可盈。"林知行说，"她是我们的第一个用户——投资人用户。她要的不是代码，是故事。故事讲好了，她才会继续投钱。"

方小满盯着他看了很久。

"你真的变了。"他说。

"哪里变了？"

"以前你只会写代码。"方小满说，"现在你会讲故事了。"

林知行没接话。

他走到白板前，拿起马克笔，在"从工程师到产品经理的第一次尝试"下面，又加了一行字：

**产品经理的第一课：先问用户要不要，再决定做不做。**

---

当晚，林知行在白板上画三个版本的功能矩阵。

画到一半，手机响了。

他看了一眼来电显示：张老板。

"喂，张总。"他接起电话。

"小林，忙不忙？"张老板的声音很热情。

"不忙。"林知行说，"您有什么事？"

"有个事想跟你聊聊。"张老板说，"我有个朋友，在长沙开连锁超市，三百多家门店。他最近在找AI库存管理的方案，试了几家都不满意。我跟他提了你，他有兴趣聊聊。"

林知行的心跳快了一拍。

"三百多家门店？"他问。

"对。"张老板说，"年营收过亿。库存管理是他的大痛点——每年因为过期、滞销、缺货，损失超过八百万。"

八百万。

林知行盯着白板上的"专业版：定制分析"，脑子里忽然转过一个念头。

如果能拿下这个客户，专业版就不是纸上谈兵了。

"张总，"他说，"您朋友什么时候方便？"

"明天我把他微信推给你。"张老板说，"你自己约时间。"

"好。"林知行说，"谢谢张总。"

"谢什么。"张老板说，"你帮我把排课系统做好了，我帮你介绍客户，应该的。"

电话挂断。

林知行盯着手机屏幕，心跳还在加速。

方小满从厨房探出头来。

"谁？"他问。

"张老板。"林知行说，"他有个朋友在长沙开连锁超市，三百多家门店，想找AI库存管理。"

方小满愣了一下。

"三百多家？"他问。

"对。"林知行说，"年营收过亿。库存损失超过八百万。"

方小满的眼睛亮了。

"这是大客户啊。"他说。

"对。"林知行说，"如果能拿下，专业版就不是纸上谈兵了。"

"专业版？"方小满问，"你不是说专业版先不卖吗？"

"先不卖，但可以先设计。"林知行说，"如果这个客户愿意做标杆，我们就有数据了。有了数据，专业版就能上线。"

方小满想了很久。

"行。"他说，"那我们先约时间。"

"好。"

林知行转身走到白板前，拿起马克笔，在三个版本的功能矩阵下面，又加了一行字：

**标杆客户：长沙连锁超市，三百多家门店，库存损失800万/年。**

然后在下面画了一个箭头，指向"专业版"。

箭头旁边写了两个字：

**机会。**

（本章完）
